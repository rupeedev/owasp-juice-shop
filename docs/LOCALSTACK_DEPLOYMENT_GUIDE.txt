WHAT IS LOCALSTACK?
================================================================================

TL;DR: LocalStack = AWS on your laptop. No AWS account needed. No costs.

LocalStack is a cloud service emulator that runs AWS services locally:
- Simulates AWS services (ECS, ECR, EFS, S3, CloudWatch, etc.)
- Runs in Docker container
- Perfect for development, testing, and training
- 100% FREE for core services
- No AWS account required
- No surprise AWS bills

‚úì Fast iteration - no internet latency
‚úì Safe environment - can't affect real AWS
‚úì Unlimited experimentation
‚úì Learn AWS without spending money

================================================================================
CONFIGURE AWS CLI FOR LOCALSTACK
================================================================================

TL;DR: Point AWS CLI to localhost instead of AWS.

STEP 1: INSTALL AWS CLI (if not installed)
-------------------------------------------
brew install awscli

Verify:
aws --version

STEP 2: CONFIGURE AWS CLI FOR LOCALSTACK
-----------------------------------------
TL;DR: Use dummy credentials, endpoint points to localhost.

# Set dummy AWS credentials (LocalStack doesn't validate these)
aws configure --profile localstack
  AWS Access Key ID: test
  AWS Secret Access Key: test
  Default region name: us-east-1
  Default output format: json

STEP 3: CREATE ALIAS FOR LOCALSTACK COMMANDS
---------------------------------------------
TL;DR: Add to ~/.zshrc or ~/.bashrc

# Add to your shell config (~/.zshrc or ~/.bashrc)
alias awslocal="aws --endpoint-url=http://localhost:4566 --profile localstack"

# Reload shell config
source ~/.zshrc  # or source ~/.bashrc

ALTERNATIVE: Use awslocal wrapper
----------------------------------
pip install awscli-local

# Now use 'awslocal' instead of 'aws'
awslocal s3 ls

================================================================================
DEPLOY JUICE SHOP ON LOCALSTACK
================================================================================

TL;DR: 5 phases - ECR, ECS cluster, task definition, service, access.

ARCHITECTURE:
-------------
Docker Container (Juice Shop)
  ‚Üì
LocalStack ECS (Fargate emulation)
  ‚Üì
LocalStack ECR (image registry)
  ‚Üì
Accessible on http://localhost:3000

PHASE 1: SETUP ECR (CONTAINER REGISTRY)
----------------------------------------
TL;DR: Create local Docker registry in LocalStack.

# Create ECR repository
awslocal ecr create-repository --repository-name juice-shop

# Get repository URI
awslocal ecr describe-repositories --repository-names juice-shop

Expected output:
{
  "repositories": [{
    "repositoryName": "juice-shop",
    "repositoryUri": "000000000000.dkr.ecr.us-east-1.localhost.localstack.cloud:4566/juice-shop"
  }]
}

Note the repositoryUri - you'll need it.

PHASE 2: BUILD AND PUSH DOCKER IMAGE
-------------------------------------
TL;DR: Build Juice Shop image and push to LocalStack ECR.

# Navigate to Juice Shop directory
cd /Users/rupeshpanwar/Documents/AI-Projects/owasp-juice-shop

# Build Docker image
docker build -t juice-shop:latest .

# Tag for LocalStack ECR
docker tag juice-shop:latest \
  000000000000.dkr.ecr.us-east-1.localhost.localstack.cloud:4566/juice-shop:latest

# Push to LocalStack ECR (no auth needed!)
docker push \
  000000000000.dkr.ecr.us-east-1.localhost.localstack.cloud:4566/juice-shop:latest

Expected: Image pushed successfully

PHASE 3: CREATE ECS CLUSTER
----------------------------
TL;DR: Create ECS cluster in LocalStack.

# Create cluster
awslocal ecs create-cluster --cluster-name juice-shop-cluster

# Verify
awslocal ecs describe-clusters --clusters juice-shop-cluster

Expected output:
{
  "clusters": [{
    "clusterName": "juice-shop-cluster",
    "status": "ACTIVE",
    "activeServicesCount": 0,
    "runningTasksCount": 0
  }]
}

PHASE 4: CREATE TASK DEFINITION
--------------------------------
TL;DR: Define how to run Juice Shop container.

Create file: juice-shop-task-local.json

{
  "family": "juice-shop",
  "networkMode": "awsvpc",
  "requiresCompatibilities": ["FARGATE"],
  "cpu": "512",
  "memory": "1024",
  "containerDefinitions": [
    {
      "name": "juice-shop",
      "image": "000000000000.dkr.ecr.us-east-1.localhost.localstack.cloud:4566/juice-shop:latest",
      "essential": true,
      "portMappings": [
        {
          "containerPort": 3000,
          "hostPort": 3000,
          "protocol": "tcp"
        }
      ],
      "environment": [
        {
          "name": "NODE_ENV",
          "value": "production"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/juice-shop",
          "awslogs-region": "us-east-1",
          "awslogs-stream-prefix": "ecs"
        }
      }
    }
  ]
}

# Register task definition
awslocal ecs register-task-definition --cli-input-json file://juice-shop-task-local.json

# Verify
awslocal ecs list-task-definitions

Expected: juice-shop:1 in list

PHASE 5: RUN JUICE SHOP TASK
-----------------------------
TL;DR: Start the container.

# Create default VPC and subnet (LocalStack auto-creates these)
awslocal ec2 describe-subnets

# Note a subnet ID from output (e.g., subnet-xxxxx)

# Run task
awslocal ecs run-task \
  --cluster juice-shop-cluster \
  --task-definition juice-shop \
  --launch-type FARGATE \
  --network-configuration "awsvpcConfiguration={subnets=[subnet-xxxxx],assignPublicIp=ENABLED}"

# Check task status
awslocal ecs list-tasks --cluster juice-shop-cluster

# Get task details
awslocal ecs describe-tasks \
  --cluster juice-shop-cluster \
  --tasks <task-arn-from-above>

Expected: Task status "RUNNING"

PHASE 6: ACCESS JUICE SHOP
---------------------------
TL;DR: Open browser to http://localhost:3000

# Wait 30 seconds for container to start

# Access Juice Shop
open http://localhost:3000

# Or test with curl
curl http://localhost:3000

Expected: Juice Shop web interface loads

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: LocalStack won't start
------------------------------
TL;DR: Check Docker is running.

# Check Docker
docker ps

# If error, start Docker Desktop
open -a Docker

# Wait for Docker to start, then:
localstack start -d

ISSUE: Port 4566 already in use
--------------------------------
TL;DR: Stop conflicting service or change port.

# Find what's using port 4566
lsof -i :4566

# Kill the process
kill -9 <PID>

# Or configure LocalStack to use different port
export LOCALSTACK_PORT=4567
localstack start -d

ISSUE: Can't access Juice Shop on localhost:3000
-------------------------------------------------
TL;DR: Container might not be forwarding port correctly.

# Option 1: Run Juice Shop directly with Docker
docker run -d -p 3000:3000 --name juice-shop-local juice-shop:latest

# Option 2: Check if LocalStack is exposing ports
# LocalStack Free version has limitations on port forwarding

# Option 3: Use Docker Compose instead
cd /Users/rupeshpanwar/Documents/AI-Projects/owasp-juice-shop
docker-compose up -d

ISSUE: Task won't start / stays in PENDING
-------------------------------------------
TL;DR: Check task definition and network config.

# Describe task to see error
awslocal ecs describe-tasks \
  --cluster juice-shop-cluster \
  --tasks <task-arn>

# Common fixes:
# 1. Ensure subnet exists
awslocal ec2 describe-subnets

# 2. Simplify task definition (remove complex configs)
# 3. Check image URI is correct
# 4. Try lower CPU/memory values

ISSUE: awslocal command not found
----------------------------------
TL;DR: Install awscli-local or use alias.

# Option 1: Install wrapper
pip install awscli-local

# Option 2: Use alias (add to ~/.zshrc)
alias awslocal="aws --endpoint-url=http://localhost:4566"

# Then reload shell
source ~/.zshrc

ISSUE: Docker build fails
--------------------------
TL;DR: Check Docker resources.

# Increase Docker Desktop resources
# Docker Desktop ‚Üí Preferences ‚Üí Resources
# - CPUs: 4+
# - Memory: 8GB+
# - Disk: 60GB+

# Clean Docker cache
docker system prune -a

# Rebuild
docker build -t juice-shop:latest .

ISSUE: LocalStack services showing "unavailable"
-------------------------------------------------
TL;DR: Wait or restart LocalStack.

# Wait 30 seconds after start
sleep 30
curl http://localhost:4566/_localstack/health | jq

# If still unavailable, restart
localstack restart

# Check logs for errors
localstack logs

LOCALSTACK VS REAL AWS
================================================================================

TL;DR: LocalStack is 95% compatible but has differences.

WHAT WORKS:
‚úì Basic ECS/Fargate operations
‚úì ECR (container registry)
‚úì S3 buckets and objects
‚úì CloudWatch Logs (basic)
‚úì IAM (simulated, no real auth)
‚úì VPC, subnets, security groups (simulated)
‚úì Lambda functions
‚úì DynamoDB
‚úì SNS/SQS

WHAT'S DIFFERENT:
‚ö† No real networking between containers
‚ö† Fargate emulation (not true serverless)
‚ö† IAM doesn't enforce real permissions
‚ö† No cost tracking (everything is free)
‚ö† Performance not identical to AWS
‚ö† Some advanced features missing
‚ö† Port mapping can be tricky

WHAT DOESN'T WORK:
‚úó True multi-AZ replication
‚úó Real AWS integration (can't talk to real AWS)
‚úó Some advanced ECS features
‚úó EKS (Kubernetes) - limited support
‚úó Complex networking scenarios
‚úó AWS Organizations
‚úó Some managed services

BEST FOR:
‚úì Learning AWS concepts
‚úì Local development
‚úì CI/CD testing
‚úì Cost-free experimentation
‚úì DevSecOps training labs
‚úì Proof of concepts

NOT IDEAL FOR:
‚úó Production-like performance testing
‚úó Complex multi-service integrations
‚úó Testing AWS-specific quirks
‚úó Network latency simulation
‚úó Real security testing

================================================================================
CLEANUP AND RESET
================================================================================

TL;DR: How to start fresh.

SOFT RESET (Keep LocalStack running):
--------------------------------------
# Delete all ECS resources
awslocal ecs list-clusters | jq -r '.clusterArns[]' | xargs -I {} awslocal ecs delete-cluster --cluster {}

# Delete all ECR repositories
awslocal ecr describe-repositories | jq -r '.repositories[].repositoryName' | xargs -I {} awslocal ecr delete-repository --repository-name {} --force

# Delete all S3 buckets
awslocal s3 ls | awk '{print $3}' | xargs -I {} awslocal s3 rb s3://{} --force

# Delete all CloudWatch log groups
awslocal logs describe-log-groups | jq -r '.logGroups[].logGroupName' | xargs -I {} awslocal logs delete-log-group --log-group-name {}

HARD RESET (Complete restart):
-------------------------------
# Stop LocalStack
localstack stop

# Remove LocalStack container
docker rm -f localstack-main

# Remove LocalStack data volume (optional, for complete reset)
docker volume rm localstack-volume

# Clean Docker images (optional)
docker system prune -a

# Start fresh
localstack start -d

UNINSTALL LOCALSTACK:
---------------------
# Stop and remove
localstack stop
docker rm -f localstack-main

# Uninstall CLI
brew uninstall localstack-cli

# Remove Docker images
docker rmi localstack/localstack

# Remove awscli-local
pip uninstall awscli-local

================================================================================
FREQUENTLY ASKED QUESTIONS
================================================================================

Q: Is LocalStack really free?
A: Yes! Core services are 100% free. Pro version adds advanced features.

Q: Can I use this for production?
A: No. LocalStack is for development and testing only. Use real AWS for production.

Q: Do I need an AWS account?
A: No! That's the beauty of LocalStack. No AWS account, no costs, no setup.

Q: How close is LocalStack to real AWS?
A: ~95% compatible for core services. Good enough for learning and development.

Q: Can I deploy Kubernetes on LocalStack?
A: Limited EKS support. For K8s, use minikube or kind instead.

Q: Will my LocalStack skills transfer to real AWS?
A: Yes! Same AWS CLI commands, same concepts, same APIs.

Q: How much disk space does LocalStack need?
A: ~5-10GB for Docker images and LocalStack. Allocate 20GB+ to be safe.

Q: Can multiple people share one LocalStack instance?
A: Not recommended. Each developer should run their own LocalStack.

Q: How do I persist data between LocalStack restarts?
A: Use Docker volumes or configure LocalStack persistence. Check docs.

Q: Can I test AWS security features?
A: Limited. IAM is simulated. Good for learning concepts, not for security testing.
